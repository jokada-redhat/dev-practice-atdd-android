# Digital Archivist UI フローレビュー 重要度評価レポート

**作成日:** 2026-03-30
**評価者:** レビュー結果を統合的に再評価

---

## 評価基準

| 重要度 | 定義 | 対応目安 |
|--------|------|---------|
| **P0 - Critical** | ユーザーが業務を遂行できない、または誤解を招く | 即時対応 |
| **P1 - High** | UXが著しく損なわれる、複数レビュワーが同一問題を指摘 | 次スプリント |
| **P2 - Medium** | 改善すべきだが業務は可能 | バックログ |
| **P3 - Low** | あれば嬉しい改善、見た目の向上 | 余裕があれば |

## 対応計画（推奨）

### Phase 1: 即時対応（P0 + P1の一部）

**工数: S〜M、1-2日**

| 対応 | 対象ID | 作業内容 |
|------|--------|---------|
| 検索バー非表示化 | P0-1 | 未実装の検索バーを `visibility="gone"` に変更 (暫定) |
| ハードコード修正 | P0-2 | strings.xml定義 + TextInputLayout.error使用 |
| 返却フロー改善 | P0-3 | 自動finish → 「続けて返却」「完了」ボタンに変更 |
| 自動遷移調整 | P1-2 | 15秒→30秒 + カウントダウン表示、2秒→ボタン操作 |

### Phase 2: 次スプリント（P1残り）

**工数: M〜L、3-5日**

| 対応 | 対象ID | 作業内容 |
|------|--------|---------|
| 用語統一 | P1-1 | 言語戦略決定 → strings.xml全面見直し |
| Material Icons移行 | P1-4 | ic_menu_* → Material Symbols に一括置換 |
| フィルタUI改善 | P1-3 | SegmentedButton or Chip化 |
| カードのアフォーダンス | P1-5 | 矢印アイコン + リップル + clickable設定 |

### Phase 3: バックログ（P2）

**工数: L、継続的改善**

| 対応 | 対象ID | 作業内容 |
|------|--------|---------|
| 検索機能実装 | P0-1(本対応) | MemberList/BookCatalog に検索ロジック接続 |
| 貸し出しフロー簡略化 | P2-1 | 確認ダイアログ化、画面統合検討 |
| アクセシビリティ対応 | P2-2 | contentDescription一括追加 |
| バリデーション改善 | P2-3 | TextWatcher + 検索ボタン |
| Navigation Component | P2-4 | 段階的移行 |
| 空の状態改善 | P2-5 | アクション提示追加 |

## レビュー横断分析

### 複数レビュワーが指摘した共通問題（重要度が高い）

| 問題 | 佐藤 | 田中 | 鈴木 | 渡辺 | 合計 |
|------|------|------|------|------|------|
| 検索バー未実装 | ✓ | | | ✓ | 2 |
| 日英混在・用語不統一 | ✓ | | ✓ | ✓ | 3 |
| 自動遷移タイミング | ✓ | | | ✓ | 2 |
| フィルタ選択状態の視認性 | | ✓ | | ✓ | 2 |

### レビュワー別の傾向

- **佐藤さん（図書館職員）**: 業務フローとの乖離を中心に指摘。特に連続処理・検索が重要。
- **田中さん（ユーザビリティ）**: アクセシビリティとエラーハンドリングを重点的に指摘。技術的に正確。
- **鈴木さん（UX）**: 情報アーキテクチャと全体設計を俯瞰して指摘。構造的な改善提案が多い。
- **渡辺さん（若者）**: 直感的な使い勝手と見た目の問題を指摘。ユーザー目線の鋭い指摘が多い。

---

## 結論

**全14件の指摘のうち、P0が3件、P1が5件と、半数以上が早急な対応が必要。**

特に「検索バーが見えるのに機能しない」(P0-1) と「日英混在の用語不統一」(P1-1) は、4人中3人が指摘しており、ユーザー体験に最も大きな影響を与えている。

Phase 1 の対応（P0 + P1の一部）を即時実施し、Phase 2 で残りのP1を次スプリントで対応することを推奨する。
